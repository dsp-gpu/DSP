# Python тесты: Алгоритм Кейпона (MVDR)

## Что такое алгоритм Кейпона?

Представь, что у тебя есть **антенная решётка** — несколько антенн, расположенных в пространстве. Каждая антенна принимает сигнал с небольшой задержкой по сравнению с соседней (потому что сигнал распространяется со скоростью света). По этой разнице задержек можно вычислить, **откуда пришёл сигнал**.

**Алгоритм Кейпона (MVDR — Minimum Variance Distortionless Response)** — это "умная" обработка антенной решётки. В отличие от простых методов, он **подавляет помехи** из других направлений, пропуская только нужный сигнал. Поэтому его называют "минимальная дисперсия без искажений".

```
Антенная решётка         Ковариационная матрица    Пространственный рельеф
   ┌─┬─┬─┬─┬─┐                                          ▲
   │ │ │ │ │ │ ──────►  R = Y·Y^H/N + μI  ──────►      │    пик = источник
   └─┴─┴─┴─┴─┘             обратить R                   │  /\
   P антенн, N отсчётов     → R⁻¹                      └─────────────► угол
```

### Математика (кратко)

```
Y     — матрица сигнала        [P × N]   P антенн, N временных отсчётов
U     — управляющие векторы    [P × M]   M направлений сканирования
μ     — параметр регуляризации (чтобы матрица не вырождалась)

R     = (1/N) * Y @ Y^H + μ*I   — ковариационная матрица [P × P]
R⁻¹   = inv(R)                   — обратная матрица (GPU: Холецкий POTRF+POTRI)

z[m] = 1 / Re(u_m^H · R⁻¹ · u_m)   — рельеф: насколько "виден" источник с направления m
```

**Управляющий вектор** `u_m` — это комплексный вектор длины P, который описывает **ожидаемые фазовые сдвиги** между антеннами, если бы сигнал пришёл из направления m. Если сигнал реально идёт из этого направления, `z[m]` будет большим. Если нет — маленьким.

## Классы тестов

### `TestCaponReference` — математика NumPy (GPU не нужен)

Проверяет **правильность Python-реализации** алгоритма Кейпона. GPU не нужен — только NumPy.

| Метод | Что проверяет |
|-------|---------------|
| `test_relief_shape` | Размер выходного массива `z` равен M |
| `test_relief_positive` | Все z[m] > 0 (физически корректно) |
| `test_relief_finite` | Нет NaN и бесконечностей |
| `test_relief_interference_suppression` | MVDR подавляет помеху: z в направлении помехи < средний z / 2 |
| `test_beamform_shape` | Адаптивное формирование луча: размер [M, N] |
| `test_beamform_finite` | Нет NaN в выходе беамформера |
| `test_regularization_singular` | При N < P (вырожденная матрица) μ > 0 предотвращает краш |
| `test_two_sources_resolved` | Два источника на ±30° различимы одновременно |

**Параметры по умолчанию:** P=8 антенн, N=64 отсчётов, M=16 направлений.

### `TestCaponGPU` — запуск C++ бинарного файла

Запускает скомпилированный бинарник `build/debian-radeon9070/GPUWorkLib` и проверяет, что C++ тесты 01-04 завершились с `PASS`. **Требует GPU + сборку проекта.**

| Метод | Что проверяет |
|-------|---------------|
| `test_all_cpp_tests_pass` | В stdout/stderr нет сообщений о FAIL |
| `test_no_exceptions` | Бинарник не падает с исключением |

### `TestCaponRealData` — реальные данные MATLAB

Тесты с **настоящими данными** антенной решётки из MATLAB. Координаты антенн — реальные физические координаты. Данные лежат в `Doc_Addition/Capon/capon_test/build/`.

| Метод | Что проверяет |
|-------|---------------|
| `test_files_available` | Все 4 файла с данными существуют |
| `test_load_coords` | Координаты x, y загружаются (≥ 85 точек) |
| `test_load_signal_dims` | Сигнал читается как матрица [P, N] комплексных чисел |
| `test_physical_relief_properties` | Рельеф для 37×37=1369 направлений: все z > 0 и конечные |
| `test_physical_relief_matches_reference_formula` | GPU-формула (1/N) и референс (без 1/N) дают одинаковую **форму** спектра (корреляция > 0.99) |
| `test_z_values_structure` | z_values.txt содержит 4 идентичных блока по 37×37 |

**Физические параметры:**
- Частота: `f0 = 3918 МГц + 3.15 МГц = 3921.15 МГц`
- Скорость света: `c = 299792458 м/с`

## Почему два рельефа не совпадают числово, но совпадают по форме?

GPU-реализация (`CaponProcessor`) считает `R = Y@Y.H / N + μI`, а эталонный ArrayFire-прототип считает `R = Y@Y.H + μI` (без деления на N).

Это эквивалентно масштабированию `μ` на `N`:

```
GPU:  R = (1/N)·Y·Y^H + μ·I
Ref:  R = Y·Y^H + μ_ref·I,  где μ_ref = μ·N
```

Ковариационные матрицы отличаются только **масштабом** (фактором N), поэтому обратные матрицы тоже отличаются на масштаб, и рельефы `z[m]` пропорциональны. Форма спектра (где пики, где провалы) **одинакова** — поэтому проверяется корреляция > 0.99.
