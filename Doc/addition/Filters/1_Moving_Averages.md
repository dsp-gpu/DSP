## 1. Зачем нужны скользящие средние в ЦОС?

Скользящее среднее (Moving Average, MA) — это **фильтр нижних частот** в рекуррентной форме. Он:
- 📉 Подавляет высокочастотный шум в сигнале
- 📈 Выделяет медленно меняющийся тренд
- ⚡ Вычисляется **рекуррентно** — каждый новый отсчёт требует O(1) операций

**В контексте нашего радара** скользящие средние применяются:
- Сглаживание амплитуды спектра (после FFT) перед поиском максимумов
- Фильтрация оценок частоты биений `f_beat` (дальность цели) по времени
- Подавление коротких выбросов от мешающих помех
- Усреднение мощности сигнала по нескольким зондированиям

**GPU-параллелизм**: Каждый луч (antenna/beam) обрабатывается **независимо** → полный параллелизм по `channels`. Все алгоритмы ниже — это O(N) работы на луч, и они запускаются параллельно для всех лучей одновременно.

## 2. SMA — Простое скользящее среднее

### Что это
**Simple Moving Average** — среднее арифметическое N последних отсчётов. Все отсчёты в окне имеют равный вес 1/N.

### Формула (прямая)
```
SMA[n] = (x[n] + x[n-1] + ... + x[n-N+1]) / N
```

### Рекуррентная формула (эффективная, O(1))
```
SMA[n] = SMA[n-1] + (x[n] - x[n-N]) / N
```
> Нужно хранить **буфер** из N предыдущих отсчётов (кольцевой буфер).

### Параметры
| Параметр | Тип | Описание |
|----------|-----|---------|
| `N` | uint32 | Размер окна (число отсчётов) |

### Характеристики
- ✅ Простейшая реализация
- ✅ Все частоты подавлены одинаково (прямоугольное окно в частотной области → sinc-функция)
- ❌ Требует хранения кольцевого буфера N отсчётов на канал
- ❌ Медленное реагирование на изменения (задержка N/2 отсчётов)
- ❌ Боковые лепестки частотной характеристики (прямоугольное окно)

### Когда использовать
- Быстрое и простое усреднение без требований к памяти на GPU
- Предварительная обработка перед FFT (при достаточном буфере)
- Когда N мало (N ≤ 64) и буфер вмещается в LDS

## 3. EMA — Экспоненциально взвешенное скользящее среднее

### Что это
**Exponential Moving Average** — рекуррентный фильтр 1-го порядка. Каждый новый отсчёт имеет вес `alpha`, предыдущее среднее — вес `(1 - alpha)`. Старые данные **экспоненциально затухают**.

### Рекуррентная формула
```
EMA[n] = alpha * x[n] + (1 - alpha) * EMA[n-1]
```

### Параметр alpha через размер окна N
```
alpha = 2 / (N + 1)     ← стандартная формула
```

| N  | alpha  | Описание |
|----|--------|---------|
| 5  | 0.333  | Быстрый отклик |
| 10 | 0.182  | Средний |
| 20 | 0.095  | Медленный |
| 50 | 0.039  | Очень медленный |

### Начальное значение
```
EMA[0] = x[0]    ← или EMA[0] = SMA первых N отсчётов
```

### Характеристики
- ✅ **Не нужен кольцевой буфер!** — только одна переменная состояния `EMA[n-1]` на канал
- ✅ Идеально для GPU — минимальное использование памяти
- ✅ Экспоненциальное затухание → нет "прямоугольных артефактов" SMA
- ❌ Не даёт точного ответа "за последние N отсчётов" (как SMA)
- ❌ Небольшая постоянная задержка (≈ 1/alpha отсчётов)

### Применение в нашем радаре
```
// Псевдокод для GPU-ядра
// channels параллельных лучей, points отсчётов в каждом
float ema_state = x[ch * points + 0];  // один float состояния на луч
for (int n = 1; n < points; n++) {
    float x_n = x[ch * points + n];
    ema_state = alpha * x_n + (1.0f - alpha) * ema_state;
    out[ch * points + n] = ema_state;
}
```
> На GPU: `ch = blockIdx.x` — каждый луч в отдельном блоке

## 4. MMA / SMMA — Сглаженное скользящее среднее

### Что это
**Modified Moving Average** (или **Smoothed MA**) — разновидность EMA с другим значением alpha. Реагирует **медленнее** EMA при том же окне N.

### Формула
```
MMA[n] = alpha * x[n] + (1 - alpha) * MMA[n-1]
где alpha = 1 / N          ← (в отличие от EMA: 2/(N+1))
```

### Сравнение EMA vs MMA (N=10)
| Фильтр | alpha (N=10) | Скорость отклика |
|--------|-------------|-----------------|
| EMA    | 0.182 (2/11) | Быстрее |
| MMA    | 0.100 (1/10) | Медленнее, плавнее |

### Рекуррентная альтернативная форма
```
MMA[n] = (MMA[n-1] * (N-1) + x[n]) / N
```

### Когда использовать
- Когда нужна **сверхплавная** кривая без резких реакций
- Сглаживание медленно меняющейся фоновой мощности

## 5. DEMA — Двойная экспоненциальная средняя

### Что это
**Double Exponential MA** — убирает задержку EMA, применяя "двойную" компенсацию. Введён Патриком Малливи (1994).

### Математика
```
EMA1[n] = EMA(x[n], alpha)         ← первая EMA от входного сигнала
EMA2[n] = EMA(EMA1[n], alpha)      ← вторая EMA от первой EMA
DEMA[n] = 2 * EMA1[n] - EMA2[n]   ← компенсация задержки
```

### Интуиция
- EMA1 имеет задержку τ
- EMA2 имеет задержку 2τ
- DEMA = 2*EMA1 - EMA2 → экстраполяция вперёд на τ → задержка ≈ 0

### Характеристики
- ✅ Вдвое меньше задержки, чем у EMA
- ✅ Лучше отслеживает быстро меняющийся сигнал
- ❌ Более чувствителен к шуму (из-за экстраполяции)
- ❌ Требует двух состояний (EMA1, EMA2) на луч

### Когда использовать
- Когда важна минимальная задержка фильтра
- Отслеживание быстрых целей

---

## 6. TEMA — Тройная экспоненциальная средняя

### Что это
**Triple Exponential MA** — расширение DEMA с ещё меньшей задержкой. Введён Патриком Малливи (1994).

### Математика
```
EMA1[n] = EMA(x[n], alpha)
EMA2[n] = EMA(EMA1[n], alpha)
EMA3[n] = EMA(EMA2[n], alpha)
TEMA[n] = 3*EMA1[n] - 3*EMA2[n] + EMA3[n]
```

### Сравнение задержек (N=10, alpha≈0.182)
| Фильтр | Задержка (отсчётов) | Чувствительность к шуму |
|--------|---------------------|------------------------|
| EMA    | ≈ 5.5               | Низкая                 |
| DEMA   | ≈ 2.7               | Средняя                |
| TEMA   | ≈ 1.4               | Высокая                |

### Характеристики
- ✅ Минимальная задержка среди EMA-семейства
- ✅ Самый "отзывчивый" фильтр из группы
- ❌ Наиболее чувствителен к шуму (тройная экстраполяция)
- ❌ Требует трёх состояний на луч

### Когда использовать
- Когда задержка фильтра критична, а шум относительно мал
- Постобработка чистых (предварительно отфильтрованных) данных

## 7. KAMA — Адаптивная скользящая средняя Кауфмана

### Что это
**Kaufman Adaptive Moving Average** — разработана Перри Кауфманом. Автоматически **адаптирует скорость реакции** в зависимости от соотношения сигнал/шум: при чистом тренде — быстрый фильтр, при зашумлённом участке — медленный.

### Параметры
```
N = 10  ← период оценки Efficiency Ratio (ER)
f = 2   ← период быстрой EMA (при ER≈1)
s = 30  ← период медленной EMA (при ER≈0)
```
> Стандартные значения: KAMA(10, 2, 30) — рекомендация Кауфмана

### Пошаговый алгоритм

#### Шаг 1: Efficiency Ratio (ER) — коэффициент эффективности
Измеряет, насколько "направленно" движется сигнал за N отсчётов.
```
Direction = |x[n] - x[n-N]|              ← общее изменение за N отсчётов
Volatility = sum(|x[i] - x[i-1]|, i=n-N+1..n) ← суммарный путь

ER = Direction / Volatility
```
- ER ≈ 1.0 → сигнал чистый тренд (нет шума)
- ER ≈ 0.0 → сигнал шумовой (нет тренда)

#### Шаг 2: Smoothing Constant (SC) — адаптивный коэффициент
```
fast_sc = 2.0f / (f + 1)    = 2/3   ≈ 0.667   (быстрая EMA)
slow_sc = 2.0f / (s + 1)    = 2/31  ≈ 0.0645  (медленная EMA)

SC = (ER * (fast_sc - slow_sc) + slow_sc)²
```
> Возведение в квадрат усиливает нелинейность: при ER→1 SC→fast², при ER→0 SC→slow²

#### Шаг 3: KAMA
```
KAMA[0] = x[0]     ← или SMA первых N отсчётов
KAMA[n] = KAMA[n-1] + SC * (x[n] - KAMA[n-1])
```

### Графически
```
ER=1.0 (тренд):  SC≈0.444 → KAMA реагирует БЫСТРО (как EMA(2))
ER=0.5 (смешан): SC≈0.187 → KAMA реагирует СРЕДНЕ
ER=0.0 (шум):    SC≈0.004 → KAMA реагирует МЕДЛЕННО (как EMA(30))
```

### Когда использовать в радаре
- Оценка доплеровской частоты/дальности цели в условиях переменной помеховой обстановки
- Когда цель то движется (чёткий тренд), то зависает (зашумлённый участок)
- Адаптивное сглаживание без ручной настройки alpha

## 9. GPU-параллелизм: схема реализации

### Принцип
Все лучи (antennas) обрабатываются **одновременно**. Каждый луч — независимая последовательность отсчётов.

```
Входные данные: [channels × points] — плоский буфер
Thread model: blockIdx.x = луч (antenna), loop по points — последовательно
```

> **Почему последовательно?** EMA — рекуррентная зависимость: EMA[n] зависит от EMA[n-1].
> Параллелизовать внутри луча сложно (scan-алгоритмы), но параллелизм по лучам (channels) полный.

---

### Вариант A: вещественный сигнал (`float`)

Применяется к вещественным данным: амплитуда, мощность, f_beat.

```cpp
// EMA для вещественного сигнала (1 поток = 1 луч)
__global__ void ema_kernel_f32(const float* __restrict__ in,
                               float* __restrict__ out,
                               unsigned int channels,
                               unsigned int points,
                               float alpha)
{
    unsigned int ch = blockIdx.x * blockDim.x + threadIdx.x;
    if (ch >= channels) return;

    const unsigned int base = ch * points;
    float state = in[base];           // EMA[0] = x[0]
    out[base] = state;

    for (unsigned int n = 1; n < points; n++) {
        state = alpha * in[base + n] + (1.0f - alpha) * state;
        out[base + n] = state;
    }
}
```

**Запуск:**
```cpp
unsigned int grid = (channels + 255) / 256;
hipModuleLaunchKernel(kernel, grid, 1, 1, 256, 1, 1, 0, stream, args, nullptr);
```

### Вариант B: комплексный сигнал (`complex<float>` / `float2`)

Применяется к IQ-данным: I и Q компоненты фильтруются независимо с одним alpha.

```cpp
struct float2_t { float x; float y; };  // Re, Im

// EMA для комплексного сигнала (1 поток = 1 луч)
__global__ void ema_kernel_cf32(const float2_t* __restrict__ in,
                                float2_t* __restrict__ out,
                                unsigned int channels,
                                unsigned int points,
                                float alpha)
{
    unsigned int ch = blockIdx.x * blockDim.x + threadIdx.x;
    if (ch >= channels) return;

    const unsigned int base = ch * points;
    float2_t state = in[base];             // EMA[0] = x[0]
    out[base] = state;

    const float one_minus_alpha = 1.0f - alpha;
    for (unsigned int n = 1; n < points; n++) {
        float2_t x = in[base + n];
        // Re и Im фильтруются одинаково — один alpha для обоих
        state.x = alpha * x.x + one_minus_alpha * state.x;
        state.y = alpha * x.y + one_minus_alpha * state.y;
        out[base + n] = state;
    }
}
```

**Размещение в памяти:**
```
Буфер: [channels × points] × sizeof(float2_t) = [channels × points × 8 байт]
ch=0: [(x0.re, x0.im), (x1.re, x1.im), ..., (xN.re, xN.im)]
ch=1: [(x0.re, x0.im), ...]
...
std::complex<float> совместим с float2_t (одинаковый layout в памяти).
```

**Запуск (аналогично float):**
```cpp
unsigned int grid = (channels + 255) / 256;
hipModuleLaunchKernel(kernel, grid, 1, 1, 256, 1, 1, 0, stream, args, nullptr);
```

### Вариант C: Prefix Scan (параллельно внутри одного луча)
Для EMA существует параллельный алгоритм через **scan** (сканирование):
- Используется `parallel prefix scan` (инклюзивный scan)
- Подходит если `points >> channels` (много отсчётов, мало лучей)
- Сложнее в реализации, для GPUWorkLib не нужен

| Условие | Тип данных | Grid / Block |
|---------|-----------|-------------|

## 10. Python-примеры для тестирования (эталон)

```python
import numpy as np

# SMA (кольцевой буфер)
def sma(x, N):
    out = np.zeros_like(x)
    buf = np.zeros(N)
    s = 0.0
    for i, v in enumerate(x):
        old = buf[i % N]
        buf[i % N] = v
        s = s + v - old
        out[i] = s / N if i >= N-1 else np.nan
    return out

# EMA: alpha = 2/(N+1)
def ema(x, N):
    alpha = 2.0 / (N + 1)
    out = np.zeros_like(x)
    out[0] = x[0]
    for i in range(1, len(x)):
        out[i] = alpha * x[i] + (1.0 - alpha) * out[i-1]
    return out

# MMA: alpha = 1/N
def mma(x, N):
    alpha = 1.0 / N
    out = np.zeros_like(x)
    out[0] = x[0]
    for i in range(1, len(x)):
        out[i] = alpha * x[i] + (1.0 - alpha) * out[i-1]
    return out

# DEMA
def dema(x, N):
    ema1 = ema(x, N)
    ema2 = ema(ema1, N)
    return 2.0 * ema1 - ema2

# TEMA
def tema(x, N):
    ema1 = ema(x, N)
    ema2 = ema(ema1, N)
    ema3 = ema(ema2, N)
    return 3.0 * ema1 - 3.0 * ema2 + ema3

# KAMA: стандартные параметры N=10, f=2, s=30
def kama(x, N=10, f=2, s=30):
    fast_sc = 2.0 / (f + 1)
    slow_sc = 2.0 / (s + 1)
    out = np.zeros_like(x)
    out[N-1] = np.mean(x[:N])  # начало = SMA первых N
    for i in range(N, len(x)):
        direction  = abs(x[i] - x[i-N])
        volatility = np.sum(np.abs(np.diff(x[i-N:i+1])))
        er = direction / volatility if volatility > 0 else 0.0
        sc = (er * (fast_sc - slow_sc) + slow_sc) ** 2
        out[i] = out[i-1] + sc * (x[i] - out[i-1])
    return out
```
