## 1. Что такое фильтр Калмана — просто о сложном

### Метафора: GPS и мёртвое счисление

Представь: ты едешь в машине с GPS. GPS иногда даёт неточные показания (шум измерения). Но у тебя есть ещё спидометр — ты **знаешь** свою скорость и можешь **предсказать**, где ты окажешься через секунду.

Фильтр Калмана — это алгоритм, который:
1. **Предсказывает** (predict) — "где я должен быть, исходя из моей модели?"
2. **Обновляет** (update) — "а вот что показало измерение, скорректируем предсказание"
3. **Доверяет** каждому источнику пропорционально его точности

Результат: оценка **лучше**, чем и голое предсказание, и голое измерение по отдельности.

### Ключевая идея
```
Оценка = Предсказание + Коэффициент_Калмана × (Измерение - Предсказание)
         ↑ модель          ↑ K: насколько верить измерению  ↑ "инновация"
```
- K → 1: полностью верим измерению (оно точное)
- K → 0: полностью верим предсказанию (измерение шумное)

## 2. Могу ли я применять фильтр Калмана в своих задачах?

### Твои задачи:
Ты занимаешься **LFM-радарной обработкой**: генерируешь зондирующие сигналы (LFM), принимаешь отражения, выполняешь Dechirp (демодуляция), FFT, ищешь максимумы спектра → получаешь **частоту биений** `f_beat`, пропорциональную дальности цели.

```
f_beat = mu * tau    где mu = chirp rate (Гц/с), tau = задержка (с) = 2*R/c
```

**Проблема**: `f_beat`, извлечённая из каждого зондирования, шумит:
- Шум квантования FFT (разрешение 1/T Гц)
- Тепловой шум приёмника
- Помехи, клатер

**Решение**: Применить фильтр Калмана для **сглаживания** последовательности `f_beat[t]` по времени → получить точную оценку дальности с малым шумом.

### ✅ Да, ты МОЖЕШЬ и ДОЛЖЕН применять фильтр Калмана!

**Конкретные применения в твоей системе**:

| Применение | Что фильтруем | Что получаем |
|-----------|--------------|--------------|
| 🎯 **Слежение за дальностью** | `f_beat[t]` (позиция пика FFT) | Плавная оценка R(t) |
| 📡 **Слежение за скоростью** | Изменение `f_beat` во времени | Оценка V = dR/dt |
| 🔊 **Подавление шума** | Амплитуда пика A[t] | Чистая оценка RCS цели |
| 🌀 **Угловое сопровождение** | Угол прихода θ[t] по антеннам | Точная угловая траектория |

## 3. Математика фильтра Калмана

### 3.1 Скалярный случай (1D) — для чайника

Для начала разберём **самый простой случай**: одна переменная состояния, одно измерение.

**Переменные**:
```
x       — истинное (неизвестное) состояние (например, f_beat или дальность R)
x̂[n|n]  — наша оценка после обработки n-го измерения
x̂[n|n-1]— предсказание на n-й момент (ещё до измерения)
z[n]    — измерение (зашумлённое)
P[n|n]  — дисперсия ошибки оценки
P[n|n-1]— дисперсия предсказания
Q       — дисперсия шума процесса (как быстро меняется x)
R       — дисперсия шума измерения (точность датчика)
K[n]    — коэффициент усиления Калмана
```

### 3.2 Два шага на каждый момент времени

#### Шаг 1: PREDICT (предсказание)
"Что должно быть, если ничего не изменилось?"
```
x̂[n|n-1] = x̂[n-1|n-1]          ← для модели с постоянной скоростью: x̂ = x̂ + v*dt
P[n|n-1]  = P[n-1|n-1] + Q      ← неопределённость растёт со временем
```

#### Шаг 2: UPDATE (обновление по измерению)
"Измерение пришло, корректируем!"
```
K[n] = P[n|n-1] / (P[n|n-1] + R)                   ← коэффициент Калмана (0..1)
x̂[n|n] = x̂[n|n-1] + K[n] * (z[n] - x̂[n|n-1])     ← обновлённая оценка
P[n|n]  = (1 - K[n]) * P[n|n-1]                     ← уменьшение неопределённости
```

### 3.3 Физический смысл коэффициента K
```
K = P_pred / (P_pred + R)

Если R >> P_pred → K → 0 → верим предсказанию (измерение шумное)
Если R << P_pred → K → 1 → верим измерению  (модель ненадёжна)
```

## 4. Параметры Q и R — как выбирать

### R — шум измерения (задаём из спецификации системы)
```
R = sigma_measure²
```
**Для нашей системы**: `R = (delta_f / sigma_snr)²`
- `delta_f = fs / N_fft` — разрешение FFT по частоте
- При SNR = 10 dB → погрешность пика ≈ 0.1..0.5 bin → R ≈ (0.3 * delta_f)²

### Q — шум процесса (задаём из физики объекта)
```
Q = sigma_process²
```
**Для нашей системы**: насколько быстро может меняться `f_beat`?
- Если цель неподвижна или медленная → Q маленькое (Q ≈ 0.01..1 Гц²)
- Если цель маневрирует → Q большое (Q ≈ 100..1000 Гц²)

### Практические правила:
| Ситуация | Q | R | Результат |
|----------|---|---|-----------|
| Q << R | Маленькое | Большое | Гладкая кривая, медленный отклик (верим модели) |
| Q >> R | Большое | Маленькое | Быстрый отклик, больше шума (верим измерению) |
| Q ≈ R | Баланс | Баланс | Оптимальный компромисс |

**Совет**: начни с `R = (FFT_bin_size)² / 12`, `Q = R / 100`, смотри результат.

## 5. Пример: фильтрация частоты биений f_beat

### Сценарий
Радар с LFM-зондированием, период повторения T_r = 1 мс, FFT из N=1024 точек.
Цель медленно движется: скорость изменения f_beat ≈ 10 Гц/с.
Шум FFT: ≈ 1 бин = fs/N Гц.

### Python-реализация (эталон для тестирования)
```python
import numpy as np
import matplotlib.pyplot as plt

def kalman_1d(measurements, Q, R, x0=None, P0=1.0):
    """
    1D скалярный фильтр Калмана.

    measurements: array — последовательность измерений z[n]
    Q:  float — дисперсия шума процесса (насколько быстро меняется состояние)
    R:  float — дисперсия шума измерения (точность датчика)
    x0: float — начальная оценка (если None = первое измерение)
    P0: float — начальная неопределённость (дисперсия)

    Returns: оценки состояния x̂[n|n]
    """
    n = len(measurements)
    x_hat = np.zeros(n)
    P = np.zeros(n)

    # Инициализация
    x_hat[0] = measurements[0] if x0 is None else x0
    P[0] = P0

    for i in range(1, n):
        # ── Predict ──
        x_pred = x_hat[i-1]          # простая модель: состояние не меняется
        P_pred = P[i-1] + Q          # неопределённость растёт

        # ── Update ──
        K = P_pred / (P_pred + R)    # коэффициент Калмана
        x_hat[i] = x_pred + K * (measurements[i] - x_pred)
        P[i]     = (1 - K) * P_pred

    return x_hat

# ── Пример: фильтрация f_beat ──
np.random.seed(42)
N_frames = 200          # количество зондирований
f_true = 1000.0         # истинная частота биений (Гц)
sigma_R = 5.0           # шум измерения (Гц) — 1 FFT бин при fs=5MHz, N=1M

# Зашумлённые измерения (FFT пики)
z = f_true + np.random.normal(0, sigma_R, N_frames)

# Фильтрация Калманом
R = sigma_R**2          # дисперсия измерения
Q = 0.1                 # дисперсия процесса (цель почти неподвижна)
x_filtered = kalman_1d(z, Q=Q, R=R)

print(f"Шум измерений:  std = {np.std(z):.2f} Гц")
print(f"После Калмана:  std = {np.std(x_filtered - f_true):.2f} Гц")
# Результат: шум уменьшается в sqrt(R/Q) ≈ 7 раз!
```

### Ожидаемый результат
```
Шум измерений:  std ≈ 5.0  Гц
После Калмана:  std ≈ 0.7  Гц    ← уменьшение в ~7 раз
```

## 6. Расширенная модель: позиция + скорость (2D state)

Для **маневрирующих** целей нужно слежение в пространстве состояний [R, V]:

### Уравнения в матричной форме
```
Состояние:     x = [R,  V]ᵀ        (дальность, скорость)
Измерение:     z = R + шум         (только дальность)

Матрица перехода:
    F = [[1, dt],                   (R[n+1] = R[n] + V[n]*dt)
         [0,  1]]                   (V[n+1] = V[n])

Матрица наблюдения:
    H = [1, 0]                      (измеряем только R)

Шум процесса (ускорение):
    Q = sigma_a² * [[dt⁴/4, dt³/2],
                    [dt³/2, dt²  ]]

Шум измерения:
    R_scalar = sigma_R²

Уравнения (матричные):
    ── Predict ──
    x̂[n|n-1] = F · x̂[n-1|n-1]
    P[n|n-1]  = F · P[n-1|n-1] · Fᵀ + Q

    ── Update ──
    S = H · P[n|n-1] · Hᵀ + R_scalar     (скаляр)
    K = P[n|n-1] · Hᵀ / S                (2×1 вектор)
    x̂[n|n] = x̂[n|n-1] + K · (z[n] - H·x̂[n|n-1])
    P[n|n]  = (I - K·H) · P[n|n-1]
```

### Python-реализация (2D state)
```python
import numpy as np

def kalman_range_velocity(measurements, dt, sigma_a, sigma_R):
    """
    Фильтр Калмана: слежение за дальностью R и скоростью V.

    measurements: array R[n] — зашумлённые измерения дальности
    dt:      float — период зондирования (с)
    sigma_a: float — стандартное отклонение ускорения (м/с²) — модель маневра
    sigma_R: float — стандартное отклонение ошибки измерения R (м)
    """
    F = np.array([[1, dt],
                  [0,  1]])
    H = np.array([[1, 0]])
    Q = sigma_a**2 * np.array([[dt**4/4, dt**3/2],
                                [dt**3/2, dt**2  ]])
    R_mat = np.array([[sigma_R**2]])

    n = len(measurements)
    x_hat = np.zeros((n, 2))  # [R, V]

    # Инициализация
    x_hat[0] = [measurements[0], 0.0]
    P = np.eye(2) * 100.0

    for i in range(1, n):
        # Predict
        x_pred = F @ x_hat[i-1]
        P_pred = F @ P @ F.T + Q

        # Update
        S = H @ P_pred @ H.T + R_mat
        K = P_pred @ H.T @ np.linalg.inv(S)
        innovation = measurements[i] - (H @ x_pred)[0]
        x_hat[i] = x_pred + K.flatten() * innovation
        P = (np.eye(2) - K @ H) @ P_pred

    return x_hat[:, 0], x_hat[:, 1]  # R_filtered, V_filtered
```

## 7. Расширенный фильтр Калмана (EKF) и UKF

### Когда нужен EKF?
Стандартный Калман — только для **линейных** систем. В радарной обработке нелинейности возникают:
- Преобразование дальность-угол в декартовы координаты: `x = R*cos(θ), y = R*sin(θ)`
- Маневрирование цели (не постоянная скорость)
- Доплеровская скорость через разностную частоту

**EKF** (Extended Kalman Filter):
- Линеаризует нелинейные функции через **матрицу Якобиана**
- `F ← Jacobian(f, x̂)`, `H ← Jacobian(h, x̂)`
- Подходит для умеренных нелинейностей

**UKF** (Unscented Kalman Filter):
- Использует **sigma-точки** для нелинейного распространения
- Точнее EKF при сильных нелинейностях
- Сложнее, но нет ручного вычисления якобиана

| Фильтр | Линейность | Точность | Сложность |
|--------|-----------|---------|-----------|
| KF     | Только линейные | Оптимальная | Минимальная |
| EKF    | Слабо нелинейные | Хорошая | Средняя |
| UKF    | Сильно нелинейные | Отличная | Высокая |

## 8. GPU-параллелизм: Калман для всех лучей одновременно

### Ключевое наблюдение
Каждый луч (antenna) имеет **независимый** фильтр Калмана:
- Своё состояние x̂[beam]
- Свою дисперсию P[beam]
- Свои измерения z[beam][t]

Это — **полный параллелизм по лучам**!

### Схема GPU-реализации

```cpp
// Псевдокод HIP/OpenCL ядра для 1D Kalman по всем лучам
// Запуск: один поток = один луч
// Grid: (num_beams, 1, 1), Block: (1, 1, 1)

__global__ void kalman_filter_beams(
    const float* z,        // измерения [num_beams × num_frames]
    float* x_hat,          // оценки    [num_beams × num_frames]
    int num_beams, int num_frames,
    float Q, float R,
    float x0, float P0)
{
    int beam = blockIdx.x;
    if (beam >= num_beams) return;

    float x = x0;  // начальная оценка
    float P = P0;  // начальная дисперсия

    for (int t = 0; t < num_frames; t++) {
        float z_n = z[beam * num_frames + t];

        // Predict
        float x_pred = x;
        float P_pred = P + Q;

        // Update
        float K = P_pred / (P_pred + R);
        x = x_pred + K * (z_n - x_pred);
        P = (1.0f - K) * P_pred;

        x_hat[beam * num_frames + t] = x;
    }
}
```

### Параметры запуска
```cpp
dim3 grid(num_beams, 1, 1);
dim3 block(1, 1, 1);
// или для AMD: block(64, 1, 1) с несколькими лучами в блоке
```

## 9. Инициализация и стабильность

### Начальные условия
```
x̂[0] = первое измерение z[0]    ← лучший выбор
P[0]  = R                         ← неопределённость = шум измерения
```
> Если `P[0]` большое (P >> R) — фильтр быстро "захватывает" измерения.
> Если `P[0]` маленькое — фильтр долго "не доверяет" измерениям.

### Численная стабильность
При длинных последовательностях матрица P может стать несимметричной (2D+):
```cpp
// После Update, симметризовать P:
P = 0.5f * (P + P.transpose());
```
